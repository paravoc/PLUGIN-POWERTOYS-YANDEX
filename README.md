# Browser Run Overlay

MVP-расширение для Яндекс Браузера и других Chromium-браузеров в стиле PowerToys Run. По горячей клавише открывает overlay поверх текущей страницы, ищет по вкладкам, закладкам, истории и умеет формировать web search pseudo-result.

## Возможности

- вызов overlay по горячей клавише `Ctrl+Shift+Space`
- работа поверх текущей страницы без автоматического открытия новой вкладки
- поиск по открытым вкладкам, закладкам и истории
- префиксы источников: `t:`, `b:`, `h:`, `w:`
- клавиатурная навигация: `ArrowUp`, `ArrowDown`, `Enter`, `Ctrl+Enter`, `Shift+Enter`, `Esc`
- минимальный предпросмотр выбранного результата
- страница настроек на `chrome.storage.local`
- Shadow DOM для изоляции UI от стилей сайта

## Установка в режиме разработчика

1. Откройте страницу расширений в браузере.
2. Включите режим разработчика.
3. Нажмите `Load unpacked` / `Загрузить распакованное расширение`.
4. Выберите папку проекта `plugin`.
5. После загрузки откройте страницу горячих клавиш расширений и при необходимости назначьте сочетание.

## Где настраивать горячую клавишу

- В Chromium обычно используется страница `chrome://extensions/shortcuts`
- Если браузер не открывает ее автоматически, вставьте адрес вручную в адресную строку
- По умолчанию в `manifest.json` задано `Ctrl+Shift+Space`

## Как протестировать

1. Откройте любой обычный сайт, например документацию или поисковую страницу.
2. Нажмите `Ctrl+Shift+Space`.
3. Убедитесь, что overlay появился поверх страницы и фокус установлен в поле ввода.
4. Введите запрос без префикса и проверьте результаты из вкладок, закладок и истории.
5. Проверьте префиксы:
   - `t: github`
   - `b: docs`
   - `h: chromium`
   - `w: manifest v3`
6. Проверьте клавиши:
   - `Enter` открывает результат согласно настройке
   - `Ctrl+Enter` открывает в новой вкладке
   - `Shift+Enter` открывает в фоне
   - `Esc` закрывает overlay
7. Откройте страницу настроек расширения и измените источник по умолчанию, тему и поведение `Enter`.

## Архитектура

- `background.js`
  Bootstrap для service worker. Подключает модульные скрипты из папки `background/` и общие утилиты из `shared/`.

- `background/`
  Отдельные слои хранения настроек, search providers и runtime-логики открытия результатов.

- `content.js`
  Bootstrap для content overlay. Загружается последним после общих модулей и UI-класса.

- `content/`
  UI overlay, clipboard helper и логика отображения/навигации.

- `overlay.css`
  Содержит стили overlay, анимации и адаптивное поведение.

- `options.html` + `options.js`
  Страница настроек MVP: источники, тема, поведение `Enter`, сохранение запросов, web search template.

- `shared/`
  Общие константы и утилиты для background, content и options.

## Ограничения MVP

- На системных страницах браузера (`chrome://`, `browser://`, магазин расширений и аналогичные) внедрение content script недоступно.
- Web search реализован как pseudo-result через URL-шаблон, без внешнего поискового API.
- Изменение самой горячей клавиши выполняется браузером, а не кодом расширения.
